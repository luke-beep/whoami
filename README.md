  > &lt;h3 id=&quot;heading-introduction-to-80211be-wi-fi-7&quot;&gt;Introduction to 802.11be &lpar;Wi-Fi 7&rpar;&lt;/h3&gt;
&lt;ul&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Overview&lt;/strong&gt;: Wi-Fi 7, or 802.11be, is the latest evolution in the IEEE 802.11 series of Wi-Fi protocols. It builds upon the existing 802.11ax &lpar;Wi-Fi 6&rpar; technology.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Goals&lt;/strong&gt;: The primary objective of Wi-Fi 7 is to provide higher data rates, improved efficiency, and lower latency. It&#39;s designed to support the increasing demand for wireless bandwidth in both consumer and enterprise environments.&lt;/p&gt;
&lt;/li&gt;
&lt;/ul&gt;
&lt;h3 id=&quot;heading-use-cases-and-applications&quot;&gt;Use Cases and Applications&lt;/h3&gt;
&lt;ul&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Smart Homes&lt;/strong&gt;: With its increased capacity and speed, Wi-Fi 7 can seamlessly handle numerous smart home devices.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Enterprise Networks&lt;/strong&gt;: Its high throughput and efficiency make it ideal for enterprise environments with heavy data usage.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Public Wi-Fi Networks&lt;/strong&gt;: Wi-Fi 7 can provide enhanced experiences in public spaces like airports, stadiums, and malls, where large numbers of users connect simultaneously.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Emerging Technologies&lt;/strong&gt;: It supports emerging technologies that require high bandwidth and low latency, such as 8K streaming, AR/VR applications, and IoT deployments.&lt;/p&gt;
&lt;/li&gt;
&lt;/ul&gt;
&lt;h3 id=&quot;heading-technical-specifications&quot;&gt;Technical Specifications&lt;/h3&gt;
&lt;ul&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Frequency Bands&lt;/strong&gt;: Wi-Fi 7 operates across the 2.4 GHz, 5 GHz, and 6 GHz frequency bands, utilizing a wider spectrum for data transmission.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Channel Bandwidth&lt;/strong&gt;: It supports channel bandwidths up to 320 MHz, significantly higher than its predecessors, allowing for more data to be transmitted simultaneously.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Data Rates&lt;/strong&gt;: The theoretical maximum data rate is expected to exceed 30 Gbps, a substantial increase from Wi-Fi 6&#39;s 9.6 Gbps.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Multi-Link Operation &lpar;MLO&rpar;&lt;/strong&gt;: This feature allows devices to transmit and receive data over multiple frequency bands simultaneously, enhancing throughput and reliability.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Modulation Techniques&lt;/strong&gt;: 802.11be uses higher-order Quadrature Amplitude Modulation &lpar;QAM&rpar;, going up to 4096-QAM, which enhances data density.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Spectrum Utilization&lt;/strong&gt;: Wi-Fi 7&#39;s ability to operate in 2.4 GHz, 5 GHz, and 6 GHz bands allows for more flexible and efficient spectrum utilization. This is particularly crucial as the 6 GHz band offers cleaner and wider channels.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Higher Order Modulation&lt;/strong&gt;: The 4096-QAM in Wi-Fi 7 allows it to pack more data into the same spectrum. This higher-order modulation translates to higher throughput but also requires a cleaner signal, which might be challenging in congested environments.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Orthogonal Frequency Division Multiple Access &lpar;OFDMA&rpar;&lt;/strong&gt;: An advancement from Wi-Fi 6, OFDMA in Wi-Fi 7 is further refined for efficient data transmission, allowing multiple users to share a channel more effectively.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Target Wake Time &lpar;TWT&rpar;&lt;/strong&gt;: This feature, which was introduced in Wi-Fi 6, is further optimized in Wi-Fi 7. TWT helps devices conserve power by scheduling when to wake up and communicate with the router.&lt;/p&gt;
&lt;/li&gt;
&lt;/ul&gt;

&lt;ul&gt;

&lt;li&gt;&lt;p&gt;&lt;strong&gt;Low Latency&lt;/strong&gt;: The protocol is designed with low latency in mind, benefiting real-time applications like gaming, virtual reality, and video conferencing.&lt;/p&gt;

&lt;li&gt;&lt;p&gt;&lt;strong&gt;Increased Spatial Streams&lt;/strong&gt;: The number of spatial streams is expected to increase in Wi-Fi 7, further boosting the data capacity and speed.&lt;/p&gt;

&lt;li&gt;&lt;p&gt;&lt;strong&gt;Increased Network Efficiency&lt;/strong&gt;: With technologies like OFDMA and MU-MIMO being further refined, Wi-Fi 7 networks are expected to handle network traffic more efficiently, reducing bottlenecks and improving overall network performance.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Ultra-Wideband Channels&lt;/strong&gt;: Wi-Fi 7&#39;s 320 MHz channel bandwidth is a significant jump from the 160 MHz in Wi-Fi 6. This enables more data to be transmitted, but also requires cleaner and less congested airwaves for optimal performance.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Improved QoS Mechanisms&lt;/strong&gt;: Quality of Service &lpar;QoS&rpar; in Wi-Fi 7 is expected to be more sophisticated, ensuring that critical applications like real-time gaming receive priority and stable bandwidth.&lt;/p&gt;
&lt;/li&gt;

&lt;li&gt;&lt;p&gt;&lt;strong&gt;Enhanced Guard Intervals&lt;/strong&gt;: Guard intervals in Wi-Fi 7 are expected to be more finely tuned, reducing overhead and improving efficiency, especially in environments with high multi-path interference.&lt;/p&gt;

- 5. **[Introduction to Space Complexity](https://tech.lukehjo.dev/introduction-to-space-complexity)** - _Posted on $date_ üåê 
  > &lt;p&gt;Space complexity refers to the amount of memory space required by an algorithm to run to completion. It measures the total memory an algorithm needs relative to the size of its input.&lt;/p&gt;
&lt;h4 id=&quot;heading-importance-in-computer-science&quot;&gt;Importance in Computer Science&lt;/h4&gt;
&lt;p&gt;Space complexity is crucial for:&lt;/p&gt;
&lt;ul&gt;
&lt;li&gt;&lt;p&gt;Understanding the memory requirements of an algorithm.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;Developing efficient algorithms, particularly in memory-constrained environments.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;Balancing between time and space requirements &lpar;time-space tradeoff&rpar;.&lt;/p&gt;
&lt;/li&gt;
&lt;/ul&gt;
&lt;h3 id=&quot;heading-understanding-space-complexity&quot;&gt;&lt;strong&gt;Understanding Space Complexity&lt;/strong&gt;&lt;/h3&gt;
&lt;h4 id=&quot;heading-components-of-space-complexity&quot;&gt;Components of Space Complexity&lt;/h4&gt;
&lt;ol&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Input Space:&lt;/strong&gt; Memory required to store input data.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Auxiliary Space:&lt;/strong&gt; Extra space or temporary space used by the algorithm during its execution.&lt;/p&gt;
&lt;/li&gt;
&lt;/ol&gt;
&lt;h4 id=&quot;heading-expressed-using-big-o-notation&quot;&gt;Expressed Using Big O Notation&lt;/h4&gt;
&lt;p&gt;Like time complexity, space complexity is expressed in Big O notation to describe the worst-case scenario. It helps in abstracting the growth rate of memory usage as the input size increases.&lt;/p&gt;
&lt;h3 id=&quot;heading-common-space-complexities&quot;&gt;&lt;strong&gt;Common Space Complexities&lt;/strong&gt;&lt;/h3&gt;
&lt;ol&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;O&lpar;1&rpar; - Constant Space:&lt;/strong&gt; The algorithm uses a fixed amount of memory space, regardless of input size.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;O&lpar;n&rpar; - Linear Space:&lt;/strong&gt; Space complexity grows linearly with the input size. Common in algorithms that require proportional space to the inputs.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;O&lpar;n&rpar; - Quadratic Space:&lt;/strong&gt; Seen in algorithms that require space proportional to the square of the input size, often due to nested data structures.&lt;/p&gt;
&lt;/li&gt;
&lt;/ol&gt;
&lt;h3 id=&quot;heading-analyzing-space-complexity&quot;&gt;&lt;strong&gt;Analyzing Space Complexity&lt;/strong&gt;&lt;/h3&gt;
&lt;h4 id=&quot;heading-steps-for-analysis&quot;&gt;Steps for Analysis&lt;/h4&gt;
&lt;ol&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Identify Variables and Data Structures:&lt;/strong&gt; Look at all the variables and data structures used in the algorithm.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Calculate Space Usage:&lt;/strong&gt; Determine how the memory usage grows with input size.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Abstract to Big O Terms:&lt;/strong&gt; Use Big O notation to generalize the space complexity.&lt;/p&gt;
&lt;/li&gt;
&lt;/ol&gt;
&lt;h4 id=&quot;heading-simplifying-space-complexity&quot;&gt;Simplifying Space Complexity&lt;/h4&gt;
&lt;ul&gt;
&lt;li&gt;Focus on the fastest-growing terms and ignore constants. For example, &lt;code&gt;3n + 10&lt;/code&gt; simplifies to &lt;code&gt;O&lpar;n&rpar;&lt;/code&gt;.&lt;/li&gt;
&lt;/ul&gt;
&lt;h3 id=&quot;heading-practical-examples&quot;&gt;&lt;strong&gt;Practical Examples&lt;/strong&gt;&lt;/h3&gt;
&lt;ul&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Dynamic Arrays:&lt;/strong&gt; Space complexity can grow linearly as the array expands to accommodate new elements, resulting in O&lpar;n&rpar; space complexity.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Recursive Algorithms:&lt;/strong&gt; The space complexity can grow linearly with the depth of the recursive calls, often O&lpar;n&rpar;.&lt;/p&gt;
&lt;/li&gt;
&lt;/ul&gt;
&lt;h3 id=&quot;heading-real-world-implications&quot;&gt;&lt;strong&gt;Real-World Implications&lt;/strong&gt;&lt;/h3&gt;
&lt;ol&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Algorithm Efficiency:&lt;/strong&gt; In environments with limited memory, choosing algorithms with lower space complexity is crucial.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;System Performance:&lt;/strong&gt; High space complexity can lead to increased paging and slower performance, especially with large datasets.&lt;/p&gt;
&lt;/li&gt;
&lt;li&gt;&lt;p&gt;&lt;strong&gt;Trade-offs with Time Complexity:&lt;/strong&gt; Sometimes, reducing time complexity might increase space complexity and vice versa.&lt;/p&gt;
&lt;/li&gt;
&lt;/ol&gt;

```csharp
        int n = data.Length;
        for (int i = 1; i < n; ++i)
        {
            int key = data[i];
```

```csharp
public static class MergeSort
{
    public static void Execute(int[] data)
    {
        int n = data.Length;
        int[] temp = new int[n];
        Sort(data, temp, 0, n - 1);
    }

    private static void Sort(int[] data, int[] temp, int left, int right)
    {
        if (left >= right)
        {
            return;
        }

        int mid = (left + right) / 2;
        Sort(data, temp, left, mid);
        Sort(data, temp, mid + 1, right);
        Merge(data, temp, left, mid, right);
    }

    private static void Merge(int[] data, int[] temp, int left, int mid, int right)
    {
        int i = left;
        int j = mid + 1;
        int k = left;

        while (i <= mid && j <= right)
        {
            if (data[i] <= data[j])
            {
                temp[k] = data[i];
                i++;
            }
            else
            {
                temp[k] = data[j];
                j++;
            }

            k++;
        }

        while (i <= mid)
        {
            temp[k] = data[i];
            i++;
            k++;
        }

        while (j <= right)
        {
            temp[k] = data[j];
            j++;
            k++;
        }

        for (k = left; k <= right; k++)
        {
            data[k] = temp[k];
        }
    }
}
```

```csharp
public static class QuickSort
{
    public static void Execute(int[] data)
    {
        int n = data.Length;
        Sort(data, 0, n - 1);
    }

    private static void Sort(int[] data, int left, int right)
    {
        if (left >= right)
        {
            return;
        }

        int pivot = data[(left + right) / 2];
        int index = Partition(data, left, right, pivot);
        Sort(data, left, index - 1);
        Sort(data, index, right);
    }

    private static int Partition(int[] data, int left, int right, int pivot)
    {
        while (left <= right)
        {
            while (data[left] < pivot)
            {
                left++;
            }

            while (data[right] > pivot)
            {
                right--;
            }

            if (left <= right)
            {
                (data[left], data[right]) = (data[right], data[left]);
                left++;
                right--;
            }
        }

        return left;
    }
}
```
